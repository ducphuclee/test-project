# Business Bootstrap — Business Member Setup

> Chỉ chạy sau khi bootstrap.md đã detect role = business.
> Có thể chạy trực tiếp `/business-bootstrap` để refresh component catalog.
>
> Giả định: developer đã chạy `/dev-bootstrap` — `.project-info/` đã có đầy đủ.

---

## Bước 1 — Xác nhận project đã bootstrap

```bash
cat .project-info/meta.md 2>/dev/null
```

Không có `meta.md` → DỪNG:
```
Project chưa được developer bootstrap.
Developer cần chạy /bootstrap trước.
Liên hệ developer rồi chạy lại.
```

---

## Bước 2 — Scan component catalog

Detect framework từ `.project-info/stack.md`, sau đó tìm components:

```bash
ls src/components/ 2>/dev/null
ls components/ 2>/dev/null
ls app/components/ 2>/dev/null
```

Đọc nhanh tên + props của 10-15 component phổ biến nhất (Button, Input, Card, Modal, Badge, Avatar...).

Tạo `.project-info/component-catalog.md`:

```markdown
# Component Catalog

> Generated by /business-bootstrap.
> Refresh: chạy `/business-bootstrap` khi có component mới.

## Available Components

### Button
- Import: `import { Button } from "@/components/ui/button"`
- Variants: default | destructive | outline | ghost | link
- Sizes: sm | default | lg | icon

### Card
- Import: `import { Card, CardHeader, CardContent } from "@/components/ui/card"`
- Usage: container cho grouped content

[tiếp tục với các components khác]
```

## Bước 3 — Confirm design system

```bash
cat .project-info/conventions/design.md 2>/dev/null
```

**Đã có `design.md`** → hiển thị tóm tắt (colors, fonts) xác nhận với business member.

**Chưa có** → hỏi:
```
Project chưa có design system. Chọn template:
[1] Minimal Light  — Clean, precise (Vercel, Notion)
[2] Dark Pro       — Dark-native (Linear, Cursor)
[3] Warm Light     — Soft, brand-friendly (Stripe, Resend)
[4] Tự mô tả
[5] Skip
```
- 1-3 → copy từ `.claude/skills/frontend-design/templates/` vào `design.md`
- 4 → hỏi thêm mô tả, điền template phù hợp
- 5 → bỏ qua

## Bước 4 — Update CLAUDE.md

Dùng Edit tool cập nhật `CLAUDE.md`:

**Thay `**Status:**`** → `**Status:** BOOTSTRAPPED_BUSINESS`

**Thay `## Default Behavior`:**
```markdown
## Default Behavior

**Bạn là trợ lý của business member.** Hỗ trợ build UI artifacts, prototypes, documents.

Khi cần code phức tạp → spawn @coder. Cần khám phá codebase → spawn @explorer.
```

**Thay `## Agents`:**
```markdown
## Agents

| Agent | Khi nào |
|-------|---------|
| `@explorer` | Tìm component, hiểu codebase |
| `@coder` | Build artifact phức tạp |
| `@spec-reviewer` | Kiểm tra artifact đúng yêu cầu |
| `@doc-writer` | Viết specs, handoff notes |
| `@solution-architect` | Product decision phức tạp |
```

**Thay `## Commands`:**
```markdown
## Commands

| Command | Dùng khi |
|---------|----------|
| `/bootstrap` | Setup lại hoặc refresh |
| `/business-bootstrap` | Refresh component catalog |
| `/design [mô tả]` | Build UI từ component catalog |
| `/design layout` | Thay bố cục |
| `/design theme` | Thay visual / màu sắc |
| `/deploy` | Deploy UI lên preview |
| `/commit` | Commit + tạo handoff |
| `/handoff` | Kết thúc session |
| `/resume` | Mở session mới |
| `/learn` | Trích xuất design patterns |
| `/prp-prd` | Tạo PRD |
| `/prp-plan` | Convert PRD thành implementation plan |
```

**Thay `## Rules & Skills`:**
```markdown
## Rules & Skills

| Khi làm | Đọc |
|---------|-----|
| Build UI / frontend | `.claude/skills/frontend-design/SKILL.md` |
| Design system audit | `.claude/skills/design-system/SKILL.md` |
| Slides / presentations | `.claude/skills/frontend-slides/SKILL.md` |
| Research | `.claude/skills/deep-research/SKILL.md` |
| Blueprint / planning | `.claude/skills/blueprint/SKILL.md` |
| Design patterns đã học | `.project-info/design-patterns.md` |
```
